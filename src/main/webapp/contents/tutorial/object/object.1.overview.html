오브젝트 다이어그램(Object Diagram) 개요

학습 시작


우리는 지금부터 오브젝트 다이어그램에 대한 학습을 시작할 것입니다.
여러분은 직접 다이어그램을 그려보고 문제에 대한 정답을 확인할 수 있는 에디터가 준비되어 있습니다.
각 다이어그램의 구성요소들이 있으며, 각 구성요소를 가져다 사용하면 됩니다.

오브젝트 다이어그램

오브젝트 다이어그램 또는 객체 다이어그램으로 불리며 클래스 다이어그램(class diagram)과 매우 유사하게 생겼습니다.
이 다이어그램은 특정한 시간의 시스템의 전체 또는 부분적인 구조를 나타냅니다.
클래스 다이어그램보다 더 구체적이며 일반적으로 클래스 다이어그램의 예를 제공하거나 또는 테스트 케이스(test case)로서의 역할을 합니다.
특정한 오브젝트 인스턴스(instance)의 집합과, 속성, 그리고 인스턴스 사이의 링크(link)를 통해 나타냅니다.

오브젝트 다이어그램을 목적과 쓰임새

오브젝트 다이어그램은 시스템의 자세한 상태를 보여주기 때문에 어떤 시점에서 오브젝트 사이의 관계를 보고 싶을 때 작성합니다.
또는 데이터 구조를 보고 싶을 때 작성합니다.
프로젝트 분석 단계에서 클래스 다이어그램을 작성하여 시스템 구조를 설명하고 클래스 다이어그램의 정확성과 완전성을 검증하는 테스트 케이스로 오브젝트 다이어그램을 작성합니다.

자, 이제 오브젝트 다이어그램에 대해서 알아보기 전에 오브젝트, 인스턴스 등 낯선 용어가 계속해서 등장하고 있습니다.
용어를 정리하고 익힌다면 오브젝트 다이어그램에 대한 이해가 훨씬 수월해질 것입니다.
그럼 바로 다음 단계에서 배워보도록 하겠습니다.
