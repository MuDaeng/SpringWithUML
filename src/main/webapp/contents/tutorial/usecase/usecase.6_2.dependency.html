의존 관계(Dependency)

의존 관계

의존 관계에는 포함 관계와 확장 관계가 있습니다. 이 두 관계는 언제, 어떻게 쓰이는지 알아보도록 하겠습니다.

포함 관계(Include)

포함 관계는 유스케이스를 수행할 때 다른 유스케이스가 반드시 선행되어야 하는 관계를 나타냅니다.
예를 들어 유튜브 영상에 좋아요를 누르려면 로그인이 되어 있어야 합니다.

또한 서로 다른 유스케이스에서 공통된 흐름이 존재할 때, 이 흐름을 별도의 유스케이스로 구성하고, 이 유스케이스를 기존의 유스케이스에서 참조하도록 설정해 놓는 관계를 의미합니다.
예를 들어 유튜브에서 좋아요와 구독을 하려면 로그인이 되어 있어야 합니다.

표기법

'포함하는 유스케이스'에서 '포함되는 유스케이스' 방향으로 점선으로 된 화살표를 그리고, «include»라고 표시를 해줍니다.

확장 관계(Extend)

확장 관계는 기존의 유스케이스를 수정하지 않고 새로운 요구사항을 추가로 표현하고자 할 때 사용합니다.
조금 더 자세하게 설명하겠습니다.
새로운 요구사항에 해당하는 이벤트 흐름을 별도의 유스케이스로 구성하여, 기존 유스케이스로부터 참조하도록 설정하는 관계를 의미합니다.
그렇기 때문에 확장 관계는 기준 유스케이스에 부가적으로 추가된 기능을 표현하기 위해 사용합니다.

예를 들어 유튜브에서 영상을 보다 자막 버튼을 눌러 영상에 자막을 띄울 수 있습니다. 기존 동영상 시청을 하는 유스케이스에서 자막이라는 별도의 유스케이스가 추가될 수 있는 것입니다.

표기법

'확장 기능 유스케이스'에서 '확장 대상 유스케이스' 방향으로 점선으로 된 화살표를 그리고, «extend»라고 표시를 해줍니다.

정리
의존 관계의 종류 의미 표기법
포함 관계 유스케이스를 수행할 때 다른 유스케이스가 반드시 선행되어야 하는 관계를 나타냅니다.
확장 관계 특정 조건에서 한 유스케이스로만 확장되는 관계입니다.
